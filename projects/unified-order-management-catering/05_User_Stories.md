# User Stories к кейсу "Единая система управления заказами кейтеринга"
## 1. Цель
Документ описывает пользовательские требования к единой системе управления заказами кейтеринга.
## 2. Акторы
- Клиент
- Менеджер
- Директор
## 3. User Stories

### US-01 Просмотр только доступных дат
**Как** Клиент
**Я хочу** видеть только доступные даты при оформлении заказа,
**Чтобы** не выбрать занятый слот.

**Epic:** Приём и обработка заявок (MVP)  
**MoSCoW:** Must have   
**RICE:** 72,0   

**Acceptance Criteria**
| ID  | Given (предусловие) | When (действие)                         | Then (результат)                                                            |
| --- | ------------------- | --------------------------------------- | --------------------------------------------------------------------------- |
| 1   | Клиент открывает форму / корзину    | Сайт запрашивает слоты у системы| Отображаются только доступные слоты, прошедшие фильтр (≥ 24 ч и количество слотов > 0) |
| 2   | Выбран один и тот же слот в календаре двумя или более клиентами    | Заказы оформляются практически одновременно| Система использует транзакционное бронирование и фиксирует слот за первым подтвержденным заказом. Второй Клиент получает уведомление "Выбранное время недоступно, измените свой выбор" |
