## Functional Requirements к кейсу «Единая система управления заказами кейтеринга»
### FR-01 Карточка заказа
Система должна:
- создавать карточку заказа автоматически при заказе с сайта;
- позволять менеджеру создавать карточку вручную;
- проверять заполнение обязательных полей;
- иметь в карточке следующие поля и хранить их содержимое:  

    | Поле | Тип данных в поле | Примечание |
    | -- | -- | -- |
    | ФИО клиента | VARCHAR (до 255 символов) | обязательное поле |
    | Номер телефона | VARCHAR (до 20 символов) | обязательное поле |
    | E-mail | VARCHAR (до 255 символов) | обязательное поле |
    | Дата и время оформления| DATETIME | самозаполняемое поле |
    | Дата и время мероприятия / доставки / самовывоза|DATETIME|обязательное поле|
    | Доставка / Самовывоз | VARCHAR (до 20 символов) | выпадающий список |
    | Адрес доставки | VARCHAR (до 500 символов) |
    | Тип мероприятия | VARCHAR |выпадающий список (с обслуживанием / без обслуживания)|
    | Комментарий клиента | VARCHAR (до 1000 символов) |
    | Текущий статус заказа | VARCHAR | автоматически заполняемое поле (см. FR-03) |

- иметь в карточке заполняемый список позиций заказа с указанием:
    - Наименование (VARCHAR)
    - Количество (INTEGER > 0)
- позволять менеджеру редактировать карточку;
- сохранять историю изменений;
- иметь кнопки: "Отправить смету", "Сгенерировать смету".

### FR-02 Корзина и управление товарами
Система должна:
- отображать список товаров;
- позволять добавлять товар в корзину;
- изменять количество товаров;
- удалять товары;
- автоматически пересчитывать итоговую сумму заказа.

### FR-03 Управление статусами заказа
Система должна:
- поддерживать статусы: "Новый", "Ожидает оплаты", "Оплачен", "Принят в работу", "В доставке", "Завершён", "Отменён";
- автоматически менять статус при событиях (оплата, подтверждение, создание карточки и т.д.);
- позволять менеджеру изменить статус вручную с указанием причины изменения статуса;
- позволять менеджеру видеть и фильтровать заказы по статусам.

### FR-04 Календарь и слоты
Система должна:
- резервировать слот времени на период ожидания оплаты;
- фиксировать слот после успешной оплаты;
- хранить календарь доступных дат;
- отображать клиенту только доступные слоты;
- обновлять в реальном времени изменения в календаре;
- поддерживать:
    - транзакционное бронирование;
    - временный резерв слота;
    - ручное закрытие / открытие дат менеджером;
- исключать возможность овербукинга.

### FR-05 Оплата и интеграция с платёжной системой
Система должна:
- перенаправлять клиента в платёжный шлюз;
- получать callback/webhook от платёжной системы;
- обрабатывать успешную и неуспешную оплату;
- обновлять статус заказа;
- инициировать возврат средств (частичный/полный).

### FR-06 Сметы
Система должна:
- хранить стоимости позиций меню;
- формировать смету на основе наименования и количества позиции из карточки заказа и цены, сохраненной в позиции заказа (при отсутствии использовать текущую цену из прайс-листа меню, исторические версии сметы сохранять с зафиксированными ценами);
- поддерживать шаблоны смет;
- сохранять последнюю отправленную версию;
- фиксировать факт и дату отправки сметы клиенту
- при необходимости конвертировать смету в PDF.

### FR-07 Уведомления
Система должна:
- отправлять уведомления клиенту и менеджеру при:
    - новом заказе
    - отправке сметы
    - получении оплаты
    - подтверждении заказа;
- отправлять уведомления менеджеру при системных ошибках, неуспехе событий;
- поддерживать e-mail/мессенджеры;
- позволять отключать шаблонные уведомления.

### FR-08 Личный кабинет клиента
Система должна:
- Создавать личный кабинет клиента после регистрации в системе.
- Давать доступ к личному кабинету авторизованному клиенту.
- Отображать и хранить историю заказов, включая:
    - Списки заказов с указанием количества, цены за шт., стоимостью позиции, суммой заказа;
    - Статусы заказов (см.FR-03), меняющихся в зависимости от происходящих событий;
    - Прикрепленные файлы (платёжные документы, сметы);
    - Дату и время оформления заказов;
    - Примечания к заказам.
- Отображать и хранить личные данные клиента:
    - ФИО;
    - Номер телефона;
    - Адреса;
    - E-mail.
- Позволять редактировать личные данные клиента.
- Если количество заказов равно 0, то система должна отображать сообщение "История пуста, оформите свой первый заказ" и кнопку "К каталогу".


## FR-09 Блок аналитики
Система должна:
- Использовать данные о заказах, хранящиеся в системе (см. FR-01).
- Позволять задать и сбросить условия фильтрации и сортировки.
- Формировать и отображать отчётность в соответствии с заданными условиями фильтрации и сортировки.
- Отображать агрегированные показатели и динамику в соответствии с заданными условиями фильтрации и сортировки.
- Позволять скачивать отчетность в формате PDF, xlsx.
- Отображать сообщения "Нет информации соответствующей условиям фильтрации", если нет данных удовлетворяющих фильтрации.
